Cloudflare R2 — Setup

All 3 apps share the same R2 bucket for media uploads. Las 3 apps comparten el mismo bucket de R2 para subir archivos.

Steps in Cloudflare Dashboard: Pasos en el Cloudflare Dashboard:
1. Go to R2 → Create bucket and name it (e.g. near-cms-media).
2. Under Settings → Public Access, enable the public URL or add a Custom Domain.
3. Go to Manage R2 API Tokens → Create API Token with Object Read & Write permission.
4. Copy the Access Key ID, Secret Access Key, and Endpoint — you'll need them for Railway.
1. Ve a R2 → Create bucket y nómbralo (ej. near-cms-media).
2. En Settings → Public Access, activa la URL pública o agrega un Custom Domain.
3. Ve a Manage R2 API Tokens → Create API Token con permiso Object Read & Write.
4. Copia el Access Key ID, Secret Access Key y Endpoint — los necesitarás en Railway.

Railway Build Config Configuración de Build en Railway

In each Railway service, set the build and start commands using the app's --filter name. En cada servicio de Railway, configura los comandos de build y start con el --filter de la app.

Build
pnpm turbo run build --filter=@near/near-ai
Start
pnpm --filter @near/near-ai run start
Release Cmd Release Cmd
pnpm --filter @near/cms-core exec prisma migrate deploy
Release CommandRelease Command runs before the app starts on each deploy and automatically applies any pending DB migrations. se ejecuta antes de que la app inicie en cada deploy y aplica automáticamente las migraciones pendientes de la base de datos.
.env.local — templateplantilla 
# Core
DATABASE_URL=postgresql://user:pass@host:5432/dbname
NEXTAUTH_URL=https://near-ai.up.railway.app
AUTH_SECRET=run: openssl rand -base64 32

# Cloudflare R2
S3_ENDPOINT=https://<ACCOUNT_ID>.r2.cloudflarestorage.com
S3_REGION=auto
S3_BUCKET=near-cms-media
S3_ACCESS_KEY_ID=your-r2-access-key-id
S3_SECRET_ACCESS_KEY=your-r2-secret-access-key
R2_PUBLIC_URL=https://pub-xxxx.r2.dev

# Email
RESEND_API_KEY=re_xxxxxxxxxxxx

# Airtable (optional)

AIRTABLE_BASE_ID=appXXXXXXXXXX
