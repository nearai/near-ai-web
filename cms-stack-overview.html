01

Why Railway Por qué Railway

The CMS is a pnpm monorepo with 3 separate Next.js apps — each app needs its own server process, its own domain, and its own database. Railway handles all of this natively. El CMS es un monorepo pnpm con 3 apps Next.js independientes — cada app necesita su propio proceso de servidor, su propio dominio y su propia base de datos. Railway maneja todo esto de forma nativa.

Monorepo Monorepo
Supports pnpm workspaces out of the box Soporta pnpm workspaces sin configuración
You point each Railway service at a different --filter and it builds only that app, sharing the Turborepo cache. Cada servicio de Railway apunta a un --filter distinto y construye solo esa app, compartiendo el caché de Turborepo.
Database Base de datos
Managed Postgres per service Postgres administrado por servicio
Each app gets its own isolated Postgres instance with one click. No external DB provider needed. Cada app obtiene su propia instancia de Postgres aislada con un click. No se necesita un proveedor externo de base de datos.
Deploys Deploys
Git-push deploys with zero config Deploys automáticos con git push
Push to main → Railway builds and deploys automatically. Migrations run before the app starts via Release Command. Push a main → Railway construye y despliega automáticamente. Las migraciones corren antes de que la app inicie vía Release Command.
Domains Dominios
Custom domains per service Dominios personalizados por servicio
Each of the 3 apps gets its own public URL — near-demo, near-ai, near-com — all running independently. Cada una de las 3 apps tiene su propia URL — near-demo, near-ai, near-com — corriendo de forma independiente.
02

Why Cloudflare R2 Por qué Cloudflare R2

The CMS editor lets users upload images and media. That content needs to live somewhere persistent, publicly accessible, and affordable. R2 is S3-compatible, so it works with the existing AWS SDK integration without any code changes. El editor del CMS permite subir imágenes y archivos. Ese contenido necesita vivir en algún lugar persistente, públicamente accesible y económico. R2 es compatible con S3, por lo que funciona con la integración existente del AWS SDK sin cambios de código.

Cost Costo
No egress fees Sin cargos por transferencia
Unlike AWS S3, Cloudflare R2 charges zero for data transfer out. You only pay for storage. A diferencia de AWS S3, Cloudflare R2 no cobra por la transferencia de datos. Solo pagas por almacenamiento.
S3-compatible Compatible S3
Works with @aws-sdk/client-s3 Funciona con @aws-sdk/client-s3
The CMS uses the AWS SDK for uploads. R2 plugs in by just changing the endpoint URL — no rewrite needed. El CMS usa el AWS SDK para subir archivos. R2 se conecta solo cambiando la URL del endpoint — sin reescribir código.
Public CDN CDN público
Files served via Cloudflare's global network Archivos servidos desde la red global de Cloudflare
Enabling public access on the bucket gives a URL that's already behind Cloudflare's CDN — fast worldwide. Activar el acceso público en el bucket da una URL que ya está detrás del CDN de Cloudflare — rápido en todo el mundo.
